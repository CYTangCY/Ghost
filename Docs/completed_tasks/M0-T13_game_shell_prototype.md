## Shell Behaviour

- `GameShellPresenter` drives a title screen (project title "Ghost") and an act-select/hub.
- `LilyDialogueFrame` shows short guidance text sourced from `ShellDialogueData` (data-driven, not
  hardcoded per screen) — reusable by future Acts.
- Ghost has a placeholder presence; Lily has a placeholder presence + dialogue panel.
- `ShellSceneNavigationButton` + `ShellSceneNames` start Act 1 via `SceneManager`.
- `ShellReturnToHubOverlay` adds a lightweight runtime "Return to Hub" button when the Act 1 scene
  loads, without touching Act 1 puzzle rules or pure logic.
- `GameShellSceneBuilder` (Editor menu `Ghost > Build Game Shell Scene`) generates the shell scene
  and registers both scenes in Build Settings — no hand-written `.unity` YAML. A shell Editor asmdef
  (`Ghost.Presentation.Shell.Editor`) scopes the builder.
