## Static UI Structure

- `Act1IntentClassificationStaticPresenter.cs` — MonoBehaviour attached to the prototype scene's root
  UI object. Serialized fields: `cardListRoot`, `intentGroupListRoot`, `cardTemplate`,
  `intentGroupTemplate`, `renderOnStart`. `RenderSampleData()` clears and renders nine sample message
  cards plus three intent group areas (`find_item`, `ask_location`, `ask_identity`) with short
  purpose descriptions, reading from `Act1IntentClassificationSampleData`. No player input.
- `Act1IntentClassificationPrototypeSceneBuilder.cs` — Editor-only (under an `Editor/` folder, so it
  is excluded from player builds). Menu `Ghost > Build Act 1 Intent Classification Prototype Scene`
  runs `BuildAct1IntentClassificationPrototypeScene()`, which creates a new scene, builds a
  placeholder UGUI canvas, wires the presenter, renders the sample data, and saves
  `Assets/Scenes/Act1IntentClassificationPrototype.unity`.
- `README.md` — manual Unity Editor setup steps for running the builder.

## Run 002 Display Fix Summary

Problem: the generated UI used an over-tall card template for nine cards in one vertical list, so the
card text rows were compressed/clipped and the cards looked like blank pale rectangles (the intent
group areas rendered correctly). Fixes: a compact fixed card layout height; explicit built-in font,
dark-contrast color, text wrapping, and min/preferred layout height on the message text; hid the
optional card-id and intent-hint rows so the message text has room; forced a layout rebuild after
rendering; and updated the scene builder to generate compact one-line card templates. Only
presentation scripts + README + docs changed — no pure logic, ProjectSettings, Packages, or
SampleScene edits.
